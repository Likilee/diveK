# Governance Checklist

Use this checklist for final BI decisions.

## 1. Brand Core Fit

- Confirm "textbook to real life" direction is explicit.
- Confirm examples feel native and contextual, not synthetic.
- Confirm learning flow stays immersive without distracting clutter.

## 2. Visual Identity Fit

- Confirm dark immersive base uses `#121212` family.
- Confirm accent colors are limited to semantic highlights.
- Confirm UI typefaces follow DiveK standards.
- Confirm layout keeps minimalist and cinematic focus.

## 3. Tone of Voice Fit

- Confirm copy sounds like a helpful local buddy.
- Confirm wording avoids dictionary definitions and rigid translator tone.
- Confirm English remains primary and easy for non-native readers.

## 4. UX Surface Fit

- Confirm each state has intent-aligned copy.
- Confirm loading, no-result, and success states feel supportive.
- Confirm tips/tooltips coach users without lecturing.

## 5. Decision Rules

- Return `approve` only when no P0 or P1 issues remain.
- Return `approve-with-fixes` when only P2 issues remain.
- Return `reject` when any P0 issue remains unresolved.
